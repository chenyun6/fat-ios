## 🧩 核心模块

### 1. fatApp.swift - 应用入口

**职责**：
- 应用生命周期管理
- 根视图路由（登录/主页切换）
- 全局状态注入

**关键代码**：
```swift
@main
struct fatApp: App {
    var body: some Scene {
        WindowGroup {
            RootView()
                .environmentObject(UserManager.shared)
        }
    }
}
```

**RootView核心逻辑**：
- 监听 `UserManager.isLoggedIn` 状态
- 自动切换 `LoginView` 和 `ContentViewButtons`
- Token过期自动刷新
- 使用多种监听机制确保状态同步：
  - `.onAppear` - 初始化状态
  - `.onChange` - 监听状态变化
  - `.onReceive($isLoggedIn)` - 订阅发布者
  - `.onReceive(objectWillChange)` - 监听对象变化

### 4. NetworkService - 网络服务层

**设计模式**: Singleton

**基础配置**：
```swift
private let baseURL = "http://localhost:8080/web/weight"
```

**核心功能**：

#### 4.1 sendCode - 发送验证码
```swift
func sendCode(phone: String) async throws -> String
```
- **请求**: POST /send-code
- **参数**: `{ "phone": "13800138000" }`
- **返回**: 成功消息字符串

#### 4.2 login - 用户登录
```swift
func login(phone: String, code: String) async throws -> LoginResponse
```
- **请求**: POST /login
- **参数**: `{ "phone": "13800138000", "code": "123456" }`
- **返回**: LoginResponse（包含userId, tokens, expireTime）
- **日志**: 详细的请求/响应日志

#### 4.3 refreshToken - 刷新Token
```swift
func refreshToken(refreshToken: String) async throws -> LoginResponse
```
- **请求**: POST /refresh-token
- **参数**: `{ "refreshToken": "xxx" }`
- **返回**: 新的LoginResponse

#### 4.4 createRecord - 创建记录
```swift
func createRecord(weightType: Int) async throws -> Int64
```
- **请求**: POST /create-record
- **参数**: `{ "userId": 0, "weightType": 1 }`
- **请求头**: `Authorization: Bearer {token}`
- **返回**: 记录ID
- **特性**:
  - 自动Token有效性检查
  - 401错误自动刷新Token并重试
  - 更新最后使用时间

**错误处理**：
```swift
enum NetworkError: LocalizedError {
    case invalidResponse    // 无效响应
    case httpError(Int)     // HTTP错误码
    case apiError(String)   // API业务错误
}
```

**请求流程**：
```
1. 构建URLRequest
2. 设置Headers (Content-Type, Authorization)
3. 编码请求体 (JSONEncoder)
4. 发送请求 (URLSession.shared.data)
5. 检查HTTP状态码
6. 解码响应 (JSONDecoder)
7. 检查业务状态 (ApiResponse.isSuccess)
8. 返回数据或抛出错误
```

**自动Token刷新机制**：
```swift
private func ensureValidToken() async throws {
    if userManager.isTokenExpired() {
        try await refreshTokenIfNeeded()
    }
}
```

### 错误码说明

| HTTP状态码 | 含义 | 处理方式 |
|-----------|------|---------|

| 400 | 请求参数错误 | 显示错误消息 |
| 401 | 未授权/Token过期 | 刷新Token或重新登录 |
| 500 | 服务器错误 | 提示稍后重试 |

## 🔄 状态管理

### Combine框架应用

#### UserManager状态发布
```swift
class UserManager: ObservableObject {
    @Published var isLoggedIn: Bool = false
    @Published var userId: Int64?
    @Published var phone: String?
}
```

#### 视图订阅
```swift
struct RootView: View {
    @ObservedObject private var userManager = UserManager.shared
    @State private var isLoggedIn: Bool = false

    var body: some View {
        Group {
            if isLoggedIn {
                ContentViewButtons()
            } else {
                LoginView()
            }
        }
        .onReceive(userManager.$isLoggedIn) { newValue in
            isLoggedIn = newValue
        }
    }
}
```

### 状态同步策略

为确保登录状态可靠更新，使用多层监听：

1. **onAppear**: 初始化状态
2. **onChange**: 监听状态变化
3. **onReceive($isLoggedIn)**: 订阅Published属性
4. **onReceive(objectWillChange)**: 监听对象级变化

这种多层机制确保即使某一层失效，其他层也能捕获状态变化。


### 请求拦截

#### Token自动注入
```swift
if let token = UserManager.shared.accessToken {
    request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")

```

#### 自动刷新Token
```swift
// 检查响应状态码
if httpResponse.statusCode == 401 {
    try await refreshTokenIfNeeded()
    // 重试原请求

}
```

### 错误处理

三层错误处理机制：

1. **NetworkError**: 网络层错误
2. **业务逻辑错误**: 根据ApiResponse.isSuccess判断
3. **UI友好提示**: 将错误转换为用户可理解的消息

```swift
catch {
    if let networkError = error as? NetworkError {
        switch networkError {
        case .apiError(let message):
            errorMessage = message
        case .httpError(let code):
            if code == 401 {
                errorMessage = "登录已过期，请重新登录"
            } else {
                errorMessage = "网络错误，请稍后重试"
            }
        default:
            errorMessage = "操作失败，请稍后重试"
        }
    }
}
```

## 🔒 安全机制

### Token管理

#### 双Token机制
- **Access Token**: 短期有效，用于API请求
- **Refresh Token**: 长期有效，用于刷新Access Token

#### 过期策略
1. **Access Token过期**: 自动使用Refresh Token刷新
2. **Refresh Token过期**: 需要重新登录
3. **7天未使用**: 自动清除登录状态

#### 安全存储
```swift
// 存储在UserDefaults（建议生产环境使用Keychain）
UserDefaults.standard.set(accessToken, forKey: accessTokenKey)
```

### 防重复操作

#### 每日一次记录
```swift
if hasRecordedToday {
    errorMessage = "今天已经记录过了，明天再来吧~"
    return
}
```

#### 倒计时防重复发送验证码
```swift
@State private var countdown = 0

Button(action: { sendCode() }) {
    Text(countdown > 0 ? "\(countdown)s" : "重发")
}
.disabled(countdown > 0)
```

### 输入验证

#### 手机号验证
```swift
.onChange(of: phone) { newValue in
    phone = String(newValue.prefix(11)).filter { $0.isNumber }
}
```
- 仅允许数字
- 最多11位

#### 验证码验证
```swift
.onChange(of: code) { newValue in
    code = String(newValue.prefix(6)).filter { $0.isNumber }
}
```
- 仅允许数字
- 最多6位

### 性能优化

#### 1. 减少视图重建
使用 `.id()` 修饰符确保视图正确切换：
```swift
ContentViewButtons()
    .id("ContentView")
```

#### 2. 主线程更新UI
所有UI更新必须在主线程：
```swift
await MainActor.run {
    userManager.saveUserInfo(...)
}
```

#### 3. 避免过度动画
控制烟花粒子数量（30-45个）避免性能问题。

#### 4. 本地缓存
使用RecordService本地缓存今日记录状态，减少网络请求。

### 常见问题

#### Q1: 登录后页面不切换？
**解决方案**: 检查UserManager的状态更新是否在主线程执行。

#### Q2: Token过期后无法自动刷新？
**解决方案**: 确保Refresh Token有效，检查refreshToken方法的实现。

#### Q3: 烟花动画卡顿？
**解决方案**: 减少粒子数量，优化动画时长。

#### Q4: 网络请求超时？
**解决方案**: 检查后端地址配置，确保网络连接正常。

#### Q5: UserDefaults数据丢失？
**解决方案**: 确保调用 `synchronize()` 方法强制同步。

## 📚 代码规范

### Swift命名规范
- **类名**: 大驼峰 (UserManager, NetworkService)
- **方法名**: 小驼峰 (saveUserInfo, sendCode)
- **变量名**: 小驼峰 (isLoggedIn, accessToken)
- **常量**: 小驼峰 (baseURL, userIdKey)

### 注释规范
```swift
// MARK: - 功能模块标题

/// 方法说明
/// - Parameters:
///   - phone: 手机号
///   - code: 验证码
/// - Returns: 登录响应
func login(phone: String, code: String) async throws -> LoginResponse
```

### 文件组织
- 每个文件一个主要类型
- 使用 MARK 分隔功能区域
- 相关类型可放在同一文件（如ApiModels）
