# 胖了吗 - 产品文档

## 📱 产品概述

**胖了吗**是一款极简风格的iOS体重记录应用，专注于帮助用户每天记录体重变化。应用采用Apple风格设计，提供优雅流畅的用户体验。

### 产品定位
- **目标用户**：关注体重管理的iOS用户
- **核心价值**：简单、直观、每日打卡式体重记录
- **设计理念**：极简主义，符合Apple设计语言

---

## ✨ 核心功能

### 1. 用户认证系统

#### 1.1 手机号验证码登录
- **登录流程**：
  1. 输入11位手机号
  2. 获取6位验证码（60秒倒计时）
  3. 输入验证码完成登录

- **用户体验**：
  - 青春活力的渐变背景色
  - 流畅的动画过渡效果
  - 友好的错误提示信息
  - 自动焦点切换（手机号→验证码）

#### 1.2 会话管理
- **Token机制**：
  - Access Token：用于API请求认证
  - Refresh Token：用于刷新过期的Access Token
  - 自动刷新：Token过期时自动刷新

- **安全策略**：
  - 7天未使用自动登出
  - Token过期自动刷新
  - 本地安全存储（UserDefaults）

---

### 2. 每日体重记录

#### 2.1 双按钮选择设计
应用提供两个直观的按钮供用户选择：

**选项1：胖了**
- 图标：上升箭头 ⬆️
- 颜色：橙色渐变
- 副标题：承认吧
- 适用场景：体重增加

**选项2：没胖**
- 图标：对勾 ✓
- 颜色：绿色渐变
- 副标题：继续保持
- 适用场景：体重保持或减少

#### 2.2 打卡规则
- **每日一次**：每天只能记录一次，防止重复打卡
- **零点重置**：每天零点后可进行新的记录
- **本地缓存**：记录状态本地保存，跨应用启动保持
- **服务器同步**：记录实时同步到云端

#### 2.3 视觉反馈系统

**成功反馈**：
1. 触觉反馈（震动）
2. Apple风格烟花动画
   - 胖了：橙/粉/红色系粒子效果
   - 没胖：绿/薄荷/青色系粒子效果
3. 成功提示消息（3秒自动消失）
4. 状态更新提示

**反馈动画特点**：
- 粒子数量：30-45个
- 动画时长：1.8秒
- 扩散距离：60-220像素
- 三种扩散模式随机切换：
  - 圆形扩散
  - 分组扩散
  - 随机扩散

**反馈提示**：
- 成功：绿色图标+文字

- 自动消失机制
- 淡入淡出+缩放动画

## 🎨 设计规范

### 颜色系统

**主色调**：
- 橙色：`#FF6666` - 胖了按钮
- 绿色：`#00C853` - 没胖按钮
- 蓝色：系统蓝 - 链接和辅助元素

**背景色**：
- 登录页：三色渐变（桃红-浅蓝-浅紫）
- 主页：灰白渐变
- 组件背景：毛玻璃材质（Material.ultraThinMaterial）

**文字颜色**：
- 主要文字：系统Primary
- 次要文字：系统Secondary
- 成功提示：绿色
- 错误提示：红色

### 字体规范

**标题层级**：
- H1：48pt Bold Rounded（主标题）
- H2：24pt Semibold（按钮标题）
- Body：17pt Regular（正文、输入框）
- Caption：15pt Regular（副标题、提示）
- Small：13pt Regular（底部说明）

**字间距**：
- 大标题：-0.5pt（紧凑）
- 正文：0.2pt（舒适）
- 小标题：0.1pt

### 圆角规范
- 输入框：16pt
- 按钮：20pt
- 图标背景：完全圆形

### 阴影规范
- 默认：黑色5%透明度，10pt模糊，(0,5)偏移
- 选中：主色20%透明度，20pt模糊，(0,10)偏移

## 🔄 用户流程

### 首次使用流程
```
打开应用
   ↓
登录页面
   ↓
输入手机号 → 获取验证码
   ↓
输入验证码 → 登录
   ↓
主页面（今日未记录）
   ↓
选择"胖了"或"没胖"
   ↓
烟花动画反馈
   ↓
显示今日已记录状态
```

### 日常使用流程
```
打开应用
   ↓
自动登录（Token有效）
   ↓
检查今日记录状态
   ├─ 未记录 → 可选择打卡
   └─ 已记录 → 显示记录结果 + 按钮禁用
```

### Token刷新流程
```
发起API请求
   ↓
检查Token是否过期
   ├─ 未过期 → 直接请求
   └─ 已过期 → 自动刷新Token → 重试请求
```

## 📊 数据管理

### 本地存储
使用UserDefaults存储以下数据：
- 用户ID
- 手机号
- Access Token
- Refresh Token
- Token过期时间
- 最后使用时间
- 今日记录日期
- 今日记录类型

### 云端同步
所有记录通过RESTful API同步到后端服务器：
- 登录认证
- 创建记录
- Token刷新
